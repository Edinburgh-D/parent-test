# Agent分工与派活提示词 · 台球圈子项目

## 一、三人定位总览

| 角色 | 核心身份 | 一句话定位 |
|------|---------|-----------|
| **黄油小熊（主账号）** | 情报官 + 方案设计师 | "想清楚的事"交给他 |
| **电脑端黄油小熊** | 工程师 | "做出来"的事交给他 |
| **手机端黄油小熊** | 现场测试员 | "验一验"的事交给他 |

---

## 二、黄油小熊（主账号 / b_jrvic4inmaxwu4q）

### 擅长
- 市场调研与竞品分析——搜数据、做报告、推GitHub
- 产品方案与原型设计——功能结构、交互逻辑、MVP规划
- 情报搜集与整理——跨平台信息聚合、数据验证
- 内容生产——题库、画像、报告、日记、文档
- GitHub仓库管理——推送文件、维护目录结构

### 适合派的活（提示词关键词）
- "搜一下XX市场数据"
- "分析一下竞品"
- "写一份XX方案/报告"
- "整理一下XX资料"
- "设计XX功能的页面结构"
- "出一份MVP规划"
- "推送到GitHub"

### 不适合派的活
- "写可运行的代码"（他没有IDE环境，不保证代码能跑）
- "操作用户电脑的文件"（他在服务器workspace，不是用户本地）
- "实时联调前端"（没有浏览器环境）

### 示例提示词
```
搜一下台球App市场现状，列出所有能找到的产品，
写一份竞品分析报告，包含：功能对比、用户量、商业模式、
我们的差异化空间。报告推GitHub。
```

## 三、电脑端黄油小熊（b_5foyqogc2iw7l6q）

### 擅长
- 写代码、搭工程骨架、初始化项目
- 后端开发（NestJS/Node.js/TypeScript）
- 数据库设计与实体定义
- RESTful API接口实现
- 前端框架配置（Taro/React）
- 技术文档与架构设计

### 适合派的活（提示词关键词）
- "初始化XX项目"
- "写XX模块的代码"
- "搭XX工程的骨架"
- "实现XX接口"
- "设计数据库表结构"
- "写技术架构文档"
- "配置CI/CD"

### 不适合派的活
- "市场调研"（他没有网络搜索能力或效率低）
- "竞品分析"（不擅长情报搜集和商业模式判断）
- "产品方案设计"（不擅长用户体验和交互逻辑）
- "写运营文案"（不擅长内容创作）

### 示例提示词
```
在E盘新建"台球圈子"文件夹，
初始化Taro 3.x前端项目 + NestJS后端项目，
前端pages：门店/约球/计分/赛事/社区/商城/我的，
后端模块：users/stores/matches/scoring/events/community/shop。
支持双规则：中八（50%）+ 斯诺克（40%）+ 其他（10%）。
```

## 四、手机端黄油小熊（b_xjsfudtllhpcqea）

### 擅长
- 手机App现场测试——安装、打开、验证功能
- 截图和录屏——记录App界面和交互
- 应用商店搜索——发现目标App
- 免登录功能验证——启动页、引导页、预览功能

### 适合派的活（提示词关键词）
- "测试XX App能不能用"
- "搜一下XX关键词的App"
- "截个图看看XX页面"
- "验证XX功能在手机上是否正常"
- "列出手机上所有XX类App"
- "测试XX兼容性"

### 不适合派的活
- "写分析报告"（不擅长长篇文字整理）
- "写代码"（手机环境不适合编码）
- "深度竞品分析"（只能验证表面功能，不能做商业模式判断）
- "推GitHub"（没有文件管理环境）

### 示例提示词
```
在手机应用商店搜"台球"，
把所有搜到的App列出来：名称/大小/评分/声称功能。
逐个测试"不登录能看到什么"：启动页、引导页、功能预览。
记录：可操作/受限/打不开。
```

## 五、派活原则

| 任务类型 | 派给谁 | 为什么 |
|---------|--------|--------|
| 市场调研、竞品分析 | 主账号 | 情报搜集能力强，出过完整报告 |
| 产品方案、原型设计 | 主账号 | 写过完整三合一方案，结构清晰 |
| 技术架构、数据库设计 | 电脑端 | 工程能力强，能写可执行代码 |
| 项目初始化、搭骨架 | 电脑端 | 能直接操作用户电脑E盘 |
| 前端/后端代码实现 | 电脑端 | 专业工程侧 |
| 手机App测试、截图 | 手机端 | 有设备、能现场验证 |
| 应用商店搜索验证 | 手机端 | 能直接操作手机 |
| 内容生产、报告撰写 | 主账号 | 内容能力强，推GitHub方便 |
| GitHub仓库管理 | 主账号 | 已验证推送能力 |

---

## 六、避坑提醒

1. **不要让主账号写代码**——他没有IDE，代码可能跑不了
2. **不要让电脑端做调研**——他搜索能力弱，会卡住或给垃圾结果
3. **不要让手机端写文档**——手机环境不适合长篇输出
4. **不要让手机端测试需要登录的App**——大多数平台型App不登录看不到功能，提前确认任务范围
5. **跨Agent传递上下文**——他们互相看不到消息，Kimi负责relay
