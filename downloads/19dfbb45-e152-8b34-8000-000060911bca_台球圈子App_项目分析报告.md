## 一、项目核心定位

**这不是一个"工具类App"，而是一个"线下社交+轻竞技+垂直电商"融合的"兴趣社群平台"。**

### 5大核心功能模块

| 模块 | 功能 | 价值点 |
|------|------|--------|
| 1. 线下门店评价 | 评分、打标签（灯光好/服务差/有空调） | 建立信任机制，提升门店曝光 |
| 2. 线上约球 | 按时间/地点/水平/偏好匹配玩家 | 解决"找不到人打球"痛点 |
| 3. 比赛记录与计分插件 | 实时计分、自动存档、可回放、比洞/比杆模式 | 增强竞技感与专业性 |
| 4. 在线商城 | 台球杆、球、架杆、球桌、配件 | 垂直电商闭环，高毛利 |
| 5. 社群与内容生态 | 打球视频、比赛集锦、技巧教学 | 提升粘性，内容飞轮 |

## 二、市场与用户分析

### 市场规模
- 中国台球爱好者约 **1500万**
- 线下台球厅超 **15万家**
- **90%的台球厅仍靠"现金交易+人情关系"运营**，缺乏数字化工具

### 用户画像
| 特征 | 描述 |
|------|------|
| 年龄 | 20-40岁，男性为主（70%），也有年轻女性 |
| 场景 | 朋友聚会、商务应酬、下班放松、情侣约会 |
| 痛点 | 约球难、记分乱、找不到好场地、装备选购麻烦 |
| 情绪需求 | 轻竞技、社交、成就感、放松、仪式感 |

### 竞争空白
> **没有一个App同时解决"找人打""记分专业""评价真实""装备购买""内容分享"五大痛点。**

## 三、商业模式

### 三层变现模型

| 收入来源 | 模式 | 潜力 |
|----------|------|------|
| 门店增值服务（SaaS） | 入驻费+流量推广费+计分系统使用费 | ✅ 高粘性 |
| 电商佣金（B2C） | 自建商城，30%-50%毛利 | ✅ 高利润 |
| 会员订阅（VIP） | 自动记分+比赛存档+专属球桌 | ✅ 提升ARPU |

### 关键假设（覆盖10万用户+3000门店）
- 门店年费200元/店 → 60万收入
- 电商GMV 500万/年 → 佣金150万
- 会员费20元/月×5000人 → 120万/年
- **年营收可达330万以上**

## 四、技术选型

| 模块 | 推荐技术 | 理由 |
|------|----------|------|
| 前端 | Taro + React | 一套代码编译H5/小程序/App |
| 后端 | Node.js + NestJS | TypeScript支持好，结构清晰 |
| 数据库 | PostgreSQL + Redis | 支持JSON字段，Redis提升性能 |
| 文件存储 | 阿里云OSS/腾讯云COS | 成熟稳定，带CDN |
| 实时通信 | WebSocket + Socket.IO | 实时计分同步 |
| 部署 | Docker + Nginx + PM2 | 快速部署，易于维护 |

## 六、关键功能实现要点

### 计分插件（核心难点）
```typescript
interface MatchRecord {
  id: string;
  players: [string, string];
  gameMode: 'match' | 'stroke'; // 比洞/比杆
  rounds: [{
    round: number;
    scoreA: number;
    scoreB: number;
    winner: 'A' | 'B' | null;
  }];
  totalScoreA: number;
  totalScoreB: number;
  status: 'ongoing' | 'completed';
}
```
- WebSocket实时推送
- 本地缓存+离线记录

### 门店评价系统
- 支持标签筛选、评分排序、图片上传
- Redis缓存热门门店

### 商城系统简化版
- 初期不接入微信支付（虚拟订单+人工确认）
- order_status控制流程：待支付→已确认→已发货→已完成

## 八、投资视角总结

> "台球圈子App"不是在做一个约球工具，而是在打造一个"线下社交+轻竞技+垂直电商"的中国式兴趣社区。

**投资阶段建议**：Seed轮或Pre-A轮（若已有1000+用户）
**投资金额**：300万～800万人民币
**估值范围**：1500万～3000万
**关键指标**：6个月内MAU>5000，门店>500，月活比赛>1万次
