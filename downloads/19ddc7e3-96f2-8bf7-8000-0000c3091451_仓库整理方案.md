# 仓库文件整理方案

## 一、当前问题

### 根目录
- 编号不连续（01、04、05...缺02、03）
- 手机端文件多版本堆叠（App手册v1.0/v2.0/v3.0）
- 技能文档分散

### 题库目录
- 重复文件多（测评题目原型v1.0和同名文件重复）
- 题库_4-6岁.md有03和04两个编号但内容一样
- JSON版本堆积（v1.0到v1.5共6个版本）
- README.md内容太简单，没有说明演进过程

## 二、整理方案

### 目录结构调整
```
personnal-assets/
├── README.md                          # 仓库总览
├── CHANGELOG.md                       # 演进记录
├── 01_情报汇总.md
├── 02_法律合规文档.md
├── 03_项目启动方案.md
├── 04_代码Prompt库.md
├── 05_设计案例参考.md
├── 06_原型预览.html
├── 07_模拟测试核心发现.md
├── 08_项目进度回顾.md
├── 技能文档/                          # 技能相关放一起
│   ├── 小程序测评工具项目启动技能_v1.0.md
│   ├── 通用技能_业务场景适配指南.md
│   └── ...
├── 手机端方案/                        # 手机端相关放一起
│   ├── App操作手册_v3.0.md            # 只保留最终版
│   ├── AI方案收集_第三轮提示词_v3.1.md # 只保留最终版
│   └── ...
└── 题库/                              # 题库目录
    ├── README.md                      # 题库目录说明+演进记录
    ├── 父母指南_题库统一版_v3.0.json   # 最终版JSON（保留）
    ├── 父母指南_完整题库收录_v3.0.md   # 最终版收录文件（保留）
    ├── 历史版本/                      # 旧版本归档
    │   ├── v1.0/
    │   ├── v1.1/
    │   └── ...
    └── 素材文件/                      # 补全题库等素材
```

### 文件保留策略
- **题库目录只保留v3.0最终版**在根目录
- 旧版本JSON（v1.0-v1.5）移到`题库/历史版本/`
- 重复文件删除（留最新的）
- 手机端文件只保留最终版（v3.0/v3.1），旧版删除

## 三、README.md内容要求

写清楚：
1. 这个仓库是干嘛的
2. 目录结构说明
3. 题库演进过程（v1.0→v3.0每版改了什么）
4. 每个文件的作用说明
5. 最终版本是哪个

## 四、执行清单

1. 写README.md和CHANGELOG.md
2. 移动旧版本JSON到历史版本目录
3. 删除重复文件
4. 整理手机端文件到子目录
5. 整理技能文档到子目录
6. 更新仓库根目录README
