# Instagram发布 · 图文匹配防错方案 v1.0

> 问题：手机端发布时用了旧图（超市玩具区）配新文案（晚饭挑食），导致图文不匹配
> 目标：从流程上杜绝"图片A+文案B"的错配

---

## 一、问题根因分析

| 环节 | 发生了什么 | 为什么 |
|------|-----------|--------|
| ChatGPT生成 | 新场景"晚饭挑食"的图片生成需要时间 | ChatGPT生图1-3分钟，手机端没等 |
| 保存环节 | 用了之前保存的"超市玩具区"旧图 | Recents相册里有旧图，选错了 |
| 发布环节 | 没检查图片和文案是否匹配 | 流程里没有"匹配检查"步骤 |
| 结果 | 图片是超市哭闹，文案是晚饭挑食 | 用户看到=图文不匹配 |

**核心问题**：手机端在执行时"图还没生成好，但流程要继续"，于是拿旧图凑合。

---

## 二、方案A：强制"一人一图"（最严格）

**规则**：
1. 每个场景**只生成一张图**
2. 生成后**立刻保存**
3. 保存后**立刻发布**
4. 发布后**删掉相册里的这张图**（或移到归档文件夹）
5. 下一个场景**必须重新生成新图**

**优点**：
- 彻底杜绝旧图复用
- 相册里永远只有当前任务的图

**缺点**：
- 如果生成失败，没有fallback
- 每次都要等生成，不能批量准备

**适用**：对图文匹配要求极高的场景（如品牌官方号）

---

## 三、方案B：图片标签化（推荐）

**规则**：
1. 保存图片时，**给图片打标签/重命名**
   - 方法：保存后立刻截图，记录图片文件名/时间戳
   - 或：保存到特定文件夹（如"育儿-超市玩具区"）
2. Instagram选图时，**按标签/文件夹选**
   - 不是看缩略图，是看文件名/时间
3. 发布前，**快速核对**：
   - 图片内容（超市/晚饭/医院？）
   - 文案关键词（玩具/蔬菜/发烧？）
   - 不匹配→停止发布

**执行步骤（手机端）**：

```
1. ChatGPT生成图片 → 放大 → Save保存
2. 【新增】记录保存时间："图片保存于 May 7 7:45 PM，场景：晚饭挑食"
3. Instagram选图时 → 找"May 7 7:45 PM"的图（不是7:38 PM的旧图）
4. 【新增】快速核对：图片里是餐桌/蔬菜？文案是晚饭/挑食？
5. 匹配 → 发布
   不匹配 → 停止，重新生成图片
```

**优点**：
- 不依赖删除旧图
- 可以批量准备多个场景
- 有明确的核对机制

**缺点**：
- 需要手机端多一步"记录时间"的动作
- 选图时需要看时间而不是凭记忆

**适用**：多场景批量发布（如一天发3条）

---

## 四、方案C：生成超时机制（技术兜底）

**规则**：
1. ChatGPT生成图片时，**设定超时时间**（如3分钟）
2. 超时未生成 → **报错停止**，不继续执行
3. 生成成功 → 再进入保存/发布流程

**执行逻辑**：

```
生成图片（开始计时）
  ↓
等待最多3分钟
  ↓
├─ 生成成功 → 保存 → 发布
└─ 超时失败 → 停止任务，报告"生成超时"
```

**优点**：
- 避免"没图硬发"的情况
- 失败原因清晰（超时）

**缺点**：
- 如果ChatGPT本身慢，任务成功率下降
- 需要等待，不能异步执行

**适用**：自动化流程的稳定性优先于速度

---

## 五、方案D：发布前人工确认（最后防线）

**规则**：
1. 手机端执行到"发布前一步"（编辑完文案，还没点Share）
2. **截图当前界面**（图片+文案同框）
3. **发给用户确认**："图片是晚饭场景，文案是晚饭挑食，确认发布？"
4. 用户回复"确认" → 点Share
5. 用户回复"不匹配" → 停止，重新生成图片

**执行流程**：

```
ChatGPT生成 → Save → Instagram选图 → 编辑文案
  ↓
【暂停】截图（图片+文案同框）
  ↓
发截图给用户
  ↓
用户确认 → Share发布
用户否决 → 重新生成
```

**优点**：
- 100%杜绝错配（最后人工把关）
- 用户有控制权

**缺点**：
- 每次发布都要等用户确认，不能全自动
- 用户不在线时任务卡住

**适用**：对内容质量要求极高的账号（如品牌号、人设号）

---

## 六、组合方案（终极推荐）

**用"方案B（标签化）+ 方案C（超时机制）+ 方案D（人工确认）的组合**

**执行流程**：

```
Step 1: ChatGPT生成图片（设定3分钟超时）
         ↓
         ├─ 超时 → 停止，报告失败
         └─ 成功 → 进入Step 2

Step 2: Save保存 + 记录时间标签（"场景：晚饭挑食，时间：7:45 PM"）

Step 3: Instagram选图 → 按时间标签选最新图（7:45 PM，不是7:38 PM）

Step 4: 编辑文案 → 快速核对（图片关键词 vs 文案关键词）
         ↓
         ├─ 不匹配 → 停止，重新生成
         └─ 匹配 → 进入Step 5

Step 5: 【人工确认】截图（图片+文案同框）→ 发给用户
         ↓
         ├─ 用户否决 → 停止，重新生成
         └─ 用户确认 → Share发布
```

**说明**：
- 方案B和C是自动化层面的防错
- 方案D是最后一道人工防线
- 如果用户信任度高，可以跳过Step 5（纯自动化）
- 如果用户信任度低，必须保留Step 5

## 七、skill文档更新

### chatgpt-image-generator（新增）

```
触发：需要生成新场景图片
步骤：
1. 输入提示词到ChatGPT
2. 开始计时（最多3分钟）
3. 等待图片生成
4. 【超时检查】超过3分钟无图 → 停止，报告"生成超时"
5. 图片出现 → 点击放大
6. 点击右上角Save按钮保存
7. 【标签记录】记录保存时间："场景：[场景名]，时间：[当前时间]"
输出：已保存的图片时间标签
```

### instagram-poster（更新）

```
触发：图片已保存，需要发布到Instagram
步骤：
1. 打开Instagram → Profile → Create New → Create new post
2. 【标签选图】按时间标签选最新保存的图片（不是Recents里随便选）
3. Next → Next → 输入文案
4. 【快速核对】图片内容关键词 vs 文案关键词
   ├─ 不匹配 → 停止，报告"图文不匹配"
   └─ 匹配 → 进入Step 5
5. 【人工确认】截图（图片+文案同框）→ 发给用户
   ├─ 用户否决 → 停止，重新生成
   └─ 用户确认 → Share发布
6. 确认帖子出现在Feed中
输出：发布成功截图
```

## 八、实施建议

**现在立刻做的**：
1. 更新skill文档，加入"标签记录"和"快速核对"步骤
2. 手机端下一次执行时，按新流程走一次验证
3. 观察是否还会出现错配

**长期优化**：
- 如果ChatGPT生成稳定（<2分钟），可以去掉超时机制
- 如果用户对自动化信任度高，可以去掉人工确认环节
- 如果批量发布需求多，强化标签化管理

---

## 九、一句话总结

**防错核心 = 标签化（知道哪张图对哪个场景）+ 核对（发布前检查匹配）+ 人工确认（最后防线）**

没有标签 = 选图靠猜 = 容易错配
没有核对 = 发了才知道错 = 事后补救
没有确认 = 全自动但不可控 = 质量风险

三个环节层层递进，根据你的信任度选择用几个。
